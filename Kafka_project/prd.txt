Capstone Project – Detailed Specification
Capstone Title: Kafka Producer-Consumer Pipeline
Business Context: As a Backend Engineer or Integration Engineer, you are responsible for building a complete event-driven data pipeline using Apache Kafka for an e-commerce platform. Your role is to implement reliable producers for order events, build scalable consumers for real-time processing, integrate with external systems using Kafka Connect, implement stream processing with Kafka Streams, and ensure the pipeline is production-ready with monitoring, security, and operational excellence. 
3.1 Capstone Scenario & Problem Statement
You are a Senior Backend Engineer at "EcommerceFlow," an online retail platform processing thousands of orders daily. The platform needs to transition from a monolithic architecture to an event-driven microservices architecture using Kafka. The system must capture order events (order placed, payment processed, order shipped, order delivered), process them in real-time for analytics and notifications, integrate with external systems (inventory management, customer notification service), and provide dashboards for business metrics. Current challenges include: tight coupling between services, lack of real-time visibility into order status, difficulty scaling individual components, no event replay capability for debugging, and manual monitoring. Your challenge is to design and implement a complete Kafka-based event streaming pipeline with reliable producers, scalable consumer groups, Kafka Connect for integration, Kafka Streams for real-time analytics, Schema Registry for data governance, monitoring with Prometheus/Grafana, and security configurations, ensuring the pipeline is fault-tolerant, scalable, and production-ready. 
3.2 Functional Requirements (12 comprehensive requirements)
Design event-driven architecture with Kafka as central event backbone for order processing
Implement reliable Kafka producers for order lifecycle events (order.created, order.paid, order.shipped, order.delivered) 
Create scalable consumer applications with consumer groups for different processing needs (analytics, notifications, inventory updates) 
Implement Avro schemas with Schema Registry for type-safe message serialization
Build Kafka Connect pipeline to integrate with external database (source connector) and notification service (sink connector) 
Develop Kafka Streams application for real-time order analytics (order count, revenue aggregation by time window) 
Implement proper error handling, dead letter queues, and retry mechanisms
Apply partitioning strategy for order events to maintain ordering guarantees
Set up monitoring with Prometheus and Grafana for cluster health and consumer lag
Implement security configurations (authentication, authorization, encryption)
Create operational documentation and runbooks for production deployment
Demonstrate end-to-end pipeline execution with complete order lifecycle
3.3 Technical Requirements (10 specifications)
Use Apache Kafka with Docker Compose or cloud deployment (Confluent Cloud/AWS MSK)
Implement producers in Java with proper error handling and retry logic
Implement consumers with manual offset commit for exactly-once semantics
Use Confluent Schema Registry with Avro for message serialization
Deploy Kafka Connect with minimum 2 connectors (1 source, 1 sink)
Build Kafka Streams application for real-time aggregations with windowing
Configure topic replication factor ≥2 for fault tolerance
Implement monitoring with Prometheus and Grafana dashboards
Apply security configurations (SSL/TLS and ACLs at minimum)
Follow Kafka best practices for production deployment (proper configurations, resource sizing)
3.4 Capstone Tasks (Hours 1-20) - 8 major phases
Architecture Design & Kafka Cluster Setup (Hours 1-3): Design event-driven architecture with Kafka topics for order lifecycle, Define topic structure (order.created, order.paid, order.shipped, order.delivered), Set up Kafka cluster with Docker Compose (3 brokers, ZooKeeper/KRaft, Schema Registry), Configure Kafka Connect and Kafka Streams environments, Create topics with appropriate partitions and replication factor, Document cluster architecture and component interactions 
Schema Design & Producer Implementation (Hours 4-6): Design Avro schemas for order events with Schema Registry, Implement producer application for order lifecycle events (Java), Configure producer for reliability (acks=all, idempotence=true), Implement partitioning strategy (partition by order_id for ordering), Add error handling, retry logic, and logging, Test producer with various order scenarios, Document producer configurations and schema evolution strategy 
Consumer Group Implementation (Hours 7-9): Build consumer application for order analytics processing, Implement consumer group with manual offset commit, Create second consumer for order notifications, Build third consumer for inventory updates, Implement deserialization with Avro and Schema Registry, Add error handling and dead letter queue (DLQ) for failed messages, Test consumer scaling (add/remove consumers), Document consumer configurations and offset management 
Kafka Connect Integration (Hours 10-12): Deploy Kafka Connect in distributed mode, Configure JDBC source connector to ingest customer data from database, Configure sink connector (file sink or HTTP sink) for notification service, Apply Single Message Transforms (SMTs) for data enrichment, Test end-to-end data flow from source to sink, Monitor connector status and troubleshoot failures, Document connector configurations and data flow 
Kafka Streams Real-Time Analytics (Hours 13-15): Build Kafka Streams application for order analytics, Implement stateless transformations (filter orders by status), Create windowed aggregations (Count orders per hour (tumbling window), Total revenue per hour, Top products ordered), Implement KStream-KTable join (orders with customer data), Write aggregated results to output topics, Test stream processing with sample data, Document topology and aggregation logic 
Monitoring & Observability (Hours 16-17): Configure Kafka metrics export with JMX, Set up Prometheus to scrape Kafka metrics, Create Grafana dashboards (Broker metrics (throughput, disk usage, leader elections), Consumer lag by consumer group, Producer throughput and latency, Kafka Connect connector status), Configure alerts for critical metrics (high consumer lag, broker down), Document monitoring setup and alert thresholds 
Security & Production Hardening (Hours 18-19): Configure SSL/TLS for broker-to-broker and client-to-broker communication, Set up SASL authentication (SASL/PLAIN or SASL/SCRAM), Create ACL policies for topics and consumer groups, Configure client quotas to prevent resource exhaustion, Test secure connections from producers and consumers, Document security configurations and access control policies 
Testing, Documentation & Demo (Hours 20): Perform end-to-end testing (Produce order events through complete lifecycle, Verify consumers process messages correctly, Validate Kafka Connect data flow, Check Kafka Streams aggregations, Monitor dashboards for metrics), Create comprehensive documentation (Architecture diagram with data flows, Deployment guide with configurations, Operational runbook for common scenarios, Security and access control documentation), Prepare demo materials (Live pipeline execution walkthrough, Monitoring dashboards showcase, Failure scenarios and recovery demonstrations), Final validation and handover preparation
